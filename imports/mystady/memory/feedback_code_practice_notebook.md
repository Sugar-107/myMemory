**规则**：用户明确要求“修改 ipynb，让帮我方便写，记住所有的教学集中在文档中”。

**Why**：

用户不是单纯想看完整代码，而是想通过手写、填空、对照验证来记住公式和 shape 规则。
如果知识只停留在聊天里，后续打开 Notebook 学习时会丢失上下文。

**How to apply**：

修改教学 Notebook 时，优先采用以下结构：

1. **全局图**：先说明这一节在整体知识里的位置，以及要验证什么。
2. **手写 shape**：先让用户写出输入、权重、偏置、中间结果的 shape。
3. **填核心代码**：只挖空真正需要理解的几行，不让用户在大量样板代码里迷路。
4. **参考答案后置**：先练习，后给答案；答案要能直接运行验证。
5. **总结表格**：把数学含义、代码表达、结果 shape 放在同一张表里。

**重要要求**：

- 用户在聊天中问出的关键理解点，要补回 Notebook 或 memory 文档，不让教学散落。
- 对初学者，代码练习不要一次挖太多空；优先挖“shape”和“核心公式”。
- RNN/GRU/Seq2Seq 相关内容尤其要强调 shape 流动。
